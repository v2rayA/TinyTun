# TinyTun

TinyTun is a Rust-based `tun2socks` project that captures traffic from a TUN interface and forwards eligible traffic through a SOCKS5 proxy, with DNS forwarding and filtering support.

## Features

- SOCKS5 forwarding for TCP flows
- DNS forwarding with per-server route mode (`direct` or `proxy`)
- IP/network/port filtering rules
- JSON configuration file with CLI overrides
- IPv4 + optional IPv6 TUN settings
- Optional automatic route setup/cleanup

## Project Status

TinyTun is functional for common SOCKS5 + DNS use cases, but it is still evolving.

- DNS forwarding is supported.
- UDP proxying is not a full generic SOCKS5 UDP implementation yet.
- Advanced TCP edge cases may still need refinement.

## Requirements

- Rust 1.70+
- Administrator/root privileges to create and manage TUN interfaces
- Supported OS: Windows, Linux, macOS
- On Windows: `wintun.dll` must be available next to `tinytun.exe` or in `PATH`

## Build

```bash
cargo build --release
```

Binary path:

- Linux/macOS: `target/release/tinytun`
- Windows: `target/release/tinytun.exe`

## Usage

### 1) Run With Config File

```bash
tinytun run --config config.json
```

On Linux/macOS, run with elevated privileges if required:

```bash
sudo ./target/release/tinytun run --config config.json
```

On Windows, start PowerShell or CMD as Administrator, then run:

```powershell
.\target\release\tinytun.exe run --config .\config.json
```

### 2) Run With CLI Overrides

```bash
tinytun run \
  --loglevel warning \
  --socks5 127.0.0.1:1080 \
  --socks5-username user \
  --socks5-password pass \
  --socks5-dns-over-socks5 true \
  --dns-listen-port 53 \
  --dns-timeout-ms 5000 \
  --dns 8.8.8.8:53 --dns-route direct \
  --dns 1.1.1.1:53 --dns-route proxy \
  --dns-hijack-enabled false \
  --dns-hijack-mark 1 \
  --dns-hijack-table-id 100 \
  --dns-hijack-capture-tcp true \
  --exclude-process chrome.exe \
  --exclude-process code \
  --skip-ip 127.0.0.1 \
  --skip-ip 198.18.0.1 \
  --skip-network 169.254.0.0/16 \
  --block-port 22 \
  --allow-port 53 \
  --linux-process-backend auto \
  --linux-ebpf-cache-path /run/tinytun-ebpf-flow-cache.json \
  --inbound-mode linux-ebpf \
  --linux-ebpf-ingress-enabled true \
  --linux-ebpf-ingress-interface eth0 \
  --linux-ebpf-bpf-object /etc/tinytun/tinytun_ingress.bpf.o \
  --linux-ebpf-bpf-section classifier/ingress \
  --linux-ebpf-skip-map-path /sys/fs/bpf/tinytun/skip_v4 \
  --linux-ebpf-skip-map-v6-path /sys/fs/bpf/tinytun/skip_v6 \
  --linux-ebpf-ingress-mark 563 \
  --linux-ebpf-ingress-table-id 233 \
  --linux-ebpf-ingress-redirect-port 15080 \
  --linux-ebpf-ingress-redirect-tcp true \
  --linux-ebpf-ingress-redirect-udp true \
  --interface tun0 \
  --ip 198.18.0.1 \
  --netmask 255.255.255.255 \
  --ipv6-mode auto \
  --mtu 1500 \
  --auto-route \
  --auto-detect-interface
```

Or pin bypass routing to a specific physical NIC:

```bash
tinytun run --config config.json --auto-route --auto-detect-interface false --default-interface "Ethernet"
```

Notes:

- `--dns` and `--dns-route` are repeatable and paired by order.
- `--skip-ip`, `--skip-network`, `--block-port`, and `--allow-port` are repeatable.
- `--loglevel` supports `debug|info|warning|error|none` (same level set as v2ray-core).
- `--exclude-process` is repeatable and matched by process executable name (case-insensitive).
- SOCKS5 auth and DNS mode can be overridden by `--socks5-username`, `--socks5-password`, and `--socks5-dns-over-socks5`.
- Process exclusion applies to TinyTun-handled flows: matched TCP/UDP destinations are switched to direct via dynamic host bypass routes when `auto_route` is enabled.
- Dynamic bypass routes are automatically reclaimed after an idle TTL to avoid route-table bloat.
- Linux process lookup backend can be configured via `linux_backend` (`auto`/`ss`/`ebpf`). `auto` tries eBPF cache first then falls back to `ss`.
- `linux_ebpf_cache_path` expects a JSON flow cache file generated by your eBPF collector with records like `{ "protocol": "tcp", "src": "198.18.0.2:50000", "dst": "1.1.1.1:443", "process_name": "chrome" }`.
- Linux eBPF ingress mode (`inbound.mode = linux-ebpf`) bypasses TUN creation and installs tc + policy-routing + NAT redirect rules directly on ingress.
- In eBPF ingress mode, `skip_ips` / `skip_networks` are installed as direct `RETURN` rules and skip-map entries so those destinations are passed through.
- eBPF ingress IPv6 path is also installed: `ip -6 rule/route`, `ip6tables` NAT PREROUTING, and IPv6 skip map updates (`skip_map_v6_path`).
- eBPF ingress mode expects your redirect target port to be served by a transparent inbound handler.
- DNS capture on TUN path uses `dns.listen_port` from config, or `--dns-listen-port` if provided.
- DNS hijack can be controlled by `--dns-hijack-enabled`, `--dns-hijack-mark`, `--dns-hijack-table-id`, and `--dns-hijack-capture-tcp`.
- You can combine `--config` with CLI flags; CLI values override config values.
- `--auto-detect-interface` and `--default-interface` are mutually exclusive when both enabled.

## Troubleshooting

- Permission errors: run as Administrator/root.
- Windows `LoadLibraryExW` / `wintun.dll` error: place `wintun.dll` next to the executable or add its directory to `PATH`.
- Linux TUN issues: ensure TUN module is loaded (`sudo modprobe tun`).
